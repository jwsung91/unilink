62 // Transmission
63 //
64 // Strategy-aware API (recommended):
65 //
66 // send_to() / broadcast()
67 // Behaviour depends on the configured backpressure strategy:
68 // BestEffort — non-blocking; drops data when the target client's queue is full.
69 // Reliable — blocks the calling thread until queue pressure is relieved,
70 // then enqueues. For broadcast(), each client is waited on
71 // sequentially.
72 //
73 // Explicit API (escape hatch):
74 //
75 // try_send_to() / try_broadcast()
76 // Always non-blocking and always drops on full queue, regardless of strategy.
77 //
78 // send_to_blocking()
79 // Always blocks until queue pressure is relieved, regardless of strategy.
