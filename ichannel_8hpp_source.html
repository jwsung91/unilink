62 // Transmission
63 //
64 // Strategy-aware API (recommended):
65 //
66 // send() / send_line()
67 // Behaviour depends on the configured backpressure strategy:
68 // BestEffort — non-blocking; drops data when the send queue is full.
69 // Reliable — blocks the calling thread until queue pressure is relieved,
70 // then enqueues. Never drops due to backpressure alone.
71 //
72 // Explicit API (escape hatch):
73 //
74 // try_send() / try_send_line()
75 // Always non-blocking and always drops on full queue, regardless of strategy.
76 // Use when you need BestEffort behaviour on a Reliable channel for a specific
77 // call (e.g. fire-and-forget heartbeat).
78 //
79 // send_blocking() / send_line_blocking()
80 // Always blocks until queue pressure is relieved, regardless of strategy.
