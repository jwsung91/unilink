77 // Transmission
78 //
79 // Strategy-aware API (recommended):
80 //
81 // send() / send_line()
82 // Behaviour depends on the configured backpressure strategy:
83 // BestEffort — non-blocking; drops data when the send queue is full.
84 // Reliable — blocks the calling thread until queue pressure is relieved,
85 // then enqueues. Never drops due to backpressure alone.
86 //
87 // Explicit API (escape hatch):
88 //
89 // try_send() / try_send_line()
90 // Always non-blocking and always drops on full queue, regardless of strategy.
91 // Use when you need BestEffort behaviour on a Reliable channel for a specific
92 // call (e.g. fire-and-forget heartbeat).
93 //
94 // send_blocking() / send_line_blocking()
95 // Always blocks until queue pressure is relieved, regardless of strategy.
